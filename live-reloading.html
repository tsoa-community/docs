# Live reloading

COMPATIBILITY NOTE

This guide requires Node.js >= 8 and will target expressopen in new window. We currently recommend using Yarnopen in new window, npm should work but was not tested. We assume your setup is similar to the one recommended for getting startedopen in new window

TIP

We will use nodemonopen in new window and ts-nodeopen in new window for live reloading, but any tool that allows us to hook into the reloading process will do. Alternatives may, i.e. be a combination of tsc -w and triggering tsoa spec-and-routes using onchangeopen in new window.

# Supercharging our developer experience with SwaggerUI ðŸš€

SwaggerUIopen in new window is a great tool to inspect our OAS and the requests our server can handle.

While other great tools like Redocopen in new window serve beautiful documentation, a major upside of using SwaggerUI during development is the instant feedback via the "Try it out" feature.

In order to make sure we always get the latest documentation during development, we will import the latest OAS file (build/swagger.json) every time someone hits that endpoint.

DANGER

For performance reasons, importing the contents of the OAS file only once after starting the application is recommended in production.

# Installing Swagger UI Express

This module allows you to serve auto-generated swagger-ui generated API docs from express, based on our OAS file. The result is living documentation for your API hosted from your API server via a route.

yarn add swagger-ui-express
yarn add -D @types/swagger-ui-express
1
2

# Exposing a /doc endpoint

Below the body-parser, let's add another handler before the call to RegisterRoutes(app)

// src/app.ts

import express, { Response as ExResponse, Request as ExRequest } from "express";
import swaggerUi from "swagger-ui-express";

// ...

app.use("/docs", swaggerUi.serve, async (_req: ExRequest, res: ExResponse) => {
  return res.send(
    swaggerUi.generateHTML(await import("../build/swagger.json"))
  );
});
